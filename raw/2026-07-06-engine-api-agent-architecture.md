# 엔진 = API 자체 에이전트 전환 + 에이전트 스텝 대시보드 관측 (2026-07-06)

## 맥락

headless Claude Code 로그인이 **무인으로 지속되지 않음**이 확정된 상태
(2026-07-06, failure-alerting 논의에서 확인). 엔진 하이브리드 결정은
"headless 인증 막힘"을 자체 에이전트(B) 전환의 driver 1번으로 명시해 두었다.

## 사용자 결정 발언

> "2개의 파이프라인이 claude -p 로 동작하는게 아니라 에이전트를 만들어야합니다.
> API 방식으로 할건데 어떤식으로 아키텍처를 가져가면 좋을까요?"

> "그리고 그 에이전트가 어떤사고를 하고 있는지 어떻게 동작하고 있는지
> 진행상황은 어떤지 대시보드에 다 출력되어야합니다."

## 논의 — 아키텍처 제안 (합의)

### 방향

- **Anthropic Messages API + tool use로 자체 에이전트 루프를 직접 구현**하고,
  기존 Data Plane(CI 러너) 안에서 돌린다.
- 이는 엔진 하이브리드 결정이 예고한 **B 경로의 driver 발동** — headless 무인
  지속 불가가 확정됐으므로 A(`claude -p`)를 건너뛰고 B로 전환 확정.
- 엔진 인터페이스(입력: 요구사항서+diff+테마+코드 / 출력: frontmatter 포함 .md
  + 검증 결과)는 그대로 유지, 뒤의 구현체만 교체.

### Anthropic Managed Agents(호스팅형) 기각

- 소스가 사내 GitLab(16.3 CE)에 있고, 매뉴얼 파이프라인은 사내 UI 테스트
  호스트를 세션 MCP로 제어해야 함 — Anthropic 클라우드 컨테이너는 사내망에
  닿지 못한다. 소스를 외부 컨테이너에 올리는 부담도 있음.
- 자체 루프는 Control/Data Plane 분리와 정확히 맞물림 — 에이전트 루프는
  Data Plane 내부 구현일 뿐, 상위 계약(①트리거/④완료보고)을 건드리지 않는다.

### 계층 구조 (3원칙)

1. **결정적인 것은 코드, 판단이 필요한 것만 에이전트** — compare API·테마
   매핑·MR 제출·sha 전진은 러너의 일반 코드 유지. 에이전트 루프는 기존
   "테마당 엔진 1회 호출" 단위를 대체: 정적=테마당 1루프, 매뉴얼=순회 세션당
   1루프. 파이프라인 전체를 하나의 거대 에이전트로 만들지 않음(비용·관측성·재시도).
2. **에이전트 런타임은 두 파이프라인이 공유하는 단일 모듈** — SDK 클라이언트,
   루프(도구 실행→결과 회신 반복), 스트리밍 수신, 관측 이벤트 방출, 토큰 집계,
   재시도를 한 번만 구현. 파이프라인별로는 **도구 세트 + 시스템 프롬프트**만
   교체: ①정적 = 읽기 전용 코드 탐색 도구(read_file·grep·diff 조회) + structured
   output으로 frontmatter 스키마 강제 / ②매뉴얼 = 세션 MCP 도구(파일전송·UI
   자동화)를 API tool로 변환 + 스크린샷 vision 입력.
3. **기존 스킬/에이전트 자산은 프롬프트로 이식** — Claude Code 스킬 4 +
   에이전트 2의 내용물을 시스템 프롬프트·도구 설명으로 이관(하이브리드 결정이
   예고한 재튜닝 비용). 대신 "서브에이전트 중첩 불가" 제약(v4 평탄화의 원인)에서
   탈출.

### 에이전트 스텝 관측 (사용자 신규 요구)

기존 관측성 계약(표준 스키마 + 가변 단위 + webhook push)의 granularity를
한 단계 아래로 확장:

| 대시보드 표시 | API 원천 |
|---|---|
| 지금 무슨 생각 중 | thinking 블록 스트림 (adaptive + display: summarized — **사고 요약**, 원문 전체 아님) |
| 지금 무슨 동작 중 | tool_use 이벤트 (도구명 + 입력) |
| 동작 결과 | tool_result (루프를 직접 돌리므로 전부 보유) |
| 진척도 | 오케스트레이터 단계(테마 N/M) + 루프 내 스텝 카운트 |
| 비용/사용량 | 응답 usage(입출력·캐시 토큰) → 이력 DB 적재 |

- 이벤트 계층 4단: `파이프라인 실행 > 단계 > 엔진 호출(테마/세션) > 에이전트
  스텝(사고/도구/결과)`. 대시보드는 접기/펼치기 드릴다운.
- 이력 DB에 스텝 로그 저장 → 실시간 표시 + 사후 감사 추적(왜 이 문서가 이렇게
  생성됐나) 동시 해결.
- `claude -p`는 블랙박스였으나 자체 루프는 모든 스텝이 관측 이벤트 — 관측성
  요구의 가장 큰 부수 이득.

### 인증 — 블로커 구조적 소멸

- 계정 아이디/패스워드 등록 → **API 키 등록**으로 교체. 대시보드 등록 UI·암호화
  저장·상태 표시 요구는 유지, 저장 대상만 변경. 러너 잡 시작 시 환경변수 주입.
- API 키는 만료·재로그인·디바이스 확인이 없어 "무인 지속" 문제 자체가 소멸.
  남는 운영 항목: 키 해지/무효(401) 감지 시 admin 이메일 알림(기존 이메일 알림
  결정에 연결).
- 러너→AI 네트워크는 이미 확인됨(뚫려 있음).
- 비용: 호출 단위 usage 토큰이 응답에 포함 → 실행당 비용 집계 가능, 비용 실측
  질문의 측정 수단 확보.
