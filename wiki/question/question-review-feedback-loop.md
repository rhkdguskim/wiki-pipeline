# ❓ 리뷰어가 고친 내용을 다음 생성이 학습하게 할까?

현재 흐름은 단방향(AI 생성 → 사람 리뷰 → 머지)이라, 리뷰어가 MR에서 반복적으로 고치는 내용이
다음 생성에 반영되지 않는다 — 같은 실수가 매 배치마다 재발할 수 있다.

- 후보 기능: 리뷰 수정 diff·코멘트를 수집해 생성 프롬프트/스타일 가이드에 되먹이는 **피드백 루프**
- 효과: 자동화 품질이 시간이 갈수록 좋아지는 **복리 효과** — 차별화 포인트
- 관련: 품질 게이트 [[decision-mr-review-gate]] (머지율 지표) · 생성 엔진 [[entity-docu-automatic]] (프롬프트·critic 반영 지점)
- 열린 부분: 되먹임 단위(전역 스타일 vs 소스별) · 자동 반영 vs 사람 큐레이션
- 블로킹 대상: 없음 (Phase 3, 운영 데이터가 쌓인 뒤)

## 방침 (2026-07-05)

**사람 큐레이션 후 반영**. 리뷰 수정 diff·코멘트를 수집해 사람이 검토한 뒤 스타일 가이드/프롬프트에 반영. 자동 전역 반영은 오탐 전파 위험으로 기각. 되먹임 단위(전역 vs 소스별)는 Phase 3 데이터로 결정.
