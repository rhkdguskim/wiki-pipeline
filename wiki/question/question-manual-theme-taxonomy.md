# ❓ 사용자/엔지니어 매뉴얼 분류 체계는?

산출 매뉴얼을 docs-hub 안에서 어떻게 조직하나.

- **독자 축** — 사용자(작업 중심 how-to) vs 엔지니어(설치·설정·배포·트러블슈팅).
- **형식 축** — 작업 how-to vs 화면/기능 레퍼런스.
- Docu-Automatic의 "테마=문서 페이지" 방식과 정합을 맞출까, 별도 분류를 둘까 → [[entity-docs-hub]].
- 이 분류가 관측 시나리오 ↔ 매뉴얼 페이지 매핑의 기준이 된다.

## ✅ 답 (2026-07-05)

**독자 축 2분할 — 사용자 메뉴얼 / 운영파트(셋업자) 셋업·트러블슈팅** → [[decision-manual-taxonomy-two-reader]]. 형식 축(how-to vs 레퍼런스)은 지금 도입 안 함(필요 시 각 독자 축 안에서 2차). 테마=페이지는 유지하되 어느 독자인지 메타로 붙임.
