# ❓ 테마 2차 확장은 언제, 어떤 우선순위로?

생성 엔진([[entity-docu-automatic]])의 1차 스코프는 4개 테마.
2차 확장 후보: 제품별 서브페이지(28+), 라이브러리, 개발 가이드, CI/CD, API & 프로토콜.

- 블로킹 대상: Phase 3 이후 (당장은 아님)
- 판단 재료: Phase 1~2 운영에서 문서 품질·비용 실측 → [[question-cost-estimation]]

## 방침 (2026-07-05)

**실측 후 우선순위 결정**. 후보(제품별 서브페이지·라이브러리·개발 가이드·CI/CD·API & 프로토콜)만 남겨두고, Phase 1~2 운영 데이터가 쌓이면 우선순위를 매김.

## 갱신 (2026-07-06) — 후보 2개 1차 승격

후보 중 **개발 가이드**(개발환경 구성 포함, `dev-guide`)와 **API & 프로토콜**(`api-protocol`, 백엔드 소스)은
실측을 기다리지 않고 **1차 스코프로 즉시 승격**됨 → [[decision-theme-scope-expansion]].
남은 후보 — 제품별 서브페이지(28+) · 라이브러리 · CI/CD — 는 기존 방침(실측 후 우선순위) 유지라 이 질문은 open 유지.
