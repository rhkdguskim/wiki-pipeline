# 엔진 인증 — 단일 계정 로그인 (2026-07-06)

## 맥락

생성 엔진(Docu-Automatic)을 CI 러너에서 `claude -p` headless로 구동해야 하는데,
Claude Code를 **로그인 방식**(아이디/패스워드로 로그인해서 사용)으로 쓰는 것이 전제.

## 논의

- 초기 요구: 여러 Claude Code 계정을 등록해두고 **라운드 로빈**으로 호출 —
  로그아웃/로그인 회전, 토큰 만료 시 다른 아이디로 **fallback**. 등록 UI/백엔드 필요.
- 정리: 다중 계정 회전은 보류하고, **MVP는 단일 계정으로 시작**한다.

## 확정

- 엔진 인증 = 단일 Claude Code 계정 로그인.
- 단일 아이디이므로 대시보드 등록 UI/백엔드에 **로그인 아이디/패스워드 설정**이 있어야 한다. (사용자 확인)
- 크리덴셜 저장 보안은 운영 단계 과제로 연기(기존 방침 유지).

## 남은 검증

- headless 로그인이 무인으로 지속되는지(토큰 갱신·재로그인·디바이스 확인) — Phase 1 블로커.
- 단일 계정 사용량 한도 = 파이프라인 처리량 상한 — PoC 실측 대상.
