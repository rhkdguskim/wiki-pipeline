## 사용자 답변 (원문 요지)

1. **질의 서빙**: "우선은 MCP 기능을 제공해서 개발자가 MCP에 붙어서 코드베이스를 빠르게 스캔할 수 있도록
   도와주도록 만들 거야." — 질의 채널 = MCP 서버 (우선). 자체 UI 등은 후순위.
2. **checkout**: "4번째 책임으로 넣는다는 게 무슨 소리지?" → 커넥터 3책임(compare/submit/auth)에 checkout을
   추가하는 방안(A) vs 러너가 직접 git clone 하는 방안(B)을 설명한 뒤 **B 채택** —
   git 프로토콜은 SCM 중립이라 추상화할 차이가 없고, 인증 토큰만 커넥터 auth에서 얻는다.
3. **형상 관리**: "직전 버전 질의 보장 방식도 필요합니다. 그럼 형상 관리가 필요하겠지? 이 부분도 고려해야 해."
   — 인덱스를 버전 스냅샷으로 관리 + 재인덱싱 중 직전 버전 서빙 + 원자 교체 요구 확정.
