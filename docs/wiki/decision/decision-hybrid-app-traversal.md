# 결정: 앱 순회는 하이브리드다

[[entity-manual-pipeline]]의 전수 순회는 **사전 정의 시나리오(결정적 뼈대)** 와
**에이전트 자율 탐색(완전 커버 보완)** 을 함께 쓴다.

## 근거

- **시나리오** — 핵심 작업 흐름(로그인 → 작업 수행 등)을 결정적·재현 가능하게 커버. 매뉴얼의 뼈대.
- **자율 탐색** — 시나리오가 정의하지 않은 화면·기능까지 훑어 "모든 기능" 커버 목표에 접근.
- 둘을 합쳐 결정성(시나리오)과 커버리지(탐색)를 동시에 얻는다.

## 기각 대안

- **시나리오 전용** — 재현성은 최고이나, 정의 안 된 기능은 통째로 누락 → "모든 기능" 목표와 배치.
- **자율 탐색 전용** — 커버리지는 넓으나 비결정적·고비용이고 핵심 흐름 품질이 들쭉날쭉.
